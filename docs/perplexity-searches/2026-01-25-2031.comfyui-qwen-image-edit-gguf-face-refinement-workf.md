### Qwen-Image-Edit GGUF Face Refinement Workflow in ComfyUI

Use **Qwen-Image-Edit GGUF** (FP8 or quantized formats) in ComfyUI for face correction by providing a **face reference image** (tight crop or headshot), a base image with a masked edit area (e.g., gray patch over the face), and optional pose/outfit references. Combine with **FaceDetailer**, **Lightning LoRAs**, and upscaling for 100% face consistency and refinement.[1][2][3]

#### Prerequisites (2025-2026 Setup)
- Install ComfyUI with **Qwen-Image-Edit GGUF nodes** (supports FP8/GGUF for efficiency; download from ComfyUI Manager or Hugging Face repos like Qwen models).[2]
- Required custom nodes: ComfyUI-Qwen-Image-Edit, FaceDetailer, IPAdapter (for LoRAs), ControlNet (pose estimation).[1][3]
- Models: Qwen-Image-Edit-2511 GGUF (20B base), face consistency LoRAs (e.g., Lightning LoRAs), VAE for encoding.[1][3][4]
- Run on GPU with ≥16GB VRAM for 4K outputs; use RunningHub for cloud if needed.[3]

#### Step-by-Step Workflow Setup for Face Correction
1. **Prepare Reference Images**:
   - **Image 1 (Base)**: Your generated/target portrait with a **gray/black patch mask** over the face area to inpaint (use Blockify/Masking nodes).[1][3]
   - **Image 2 (Face Reference)**: Tight **face close-up or headshot** for identity (crop to face/hair; angle influences output).[1][3]
   - **Image 3 (Optional Pose/Outfit)**: Full-body reference for pose, dress, or scene control.[1]
   - Pre-process: Crop/resize head (e.g., 1024x1024), remove background, adjust brightness for consistency.[3]

2. **Qwen-Image-Edit Node Setup**:
   - Load **Qwen-Image-Edit GGUF model** (select FP8/GGUF variant in dropdown).[2]
   - Input images via multi-image slots: Base (masked), Face ref, Pose ref.[1][3]
   - **Instruction Prompt** (key for face preservation):
     ```
     A portrait of the woman in image 2 [face ref], wearing the dress/pose in image 3. Keep all facial features, eye color, proportions, expression from image 2. Remove/replace the gray patch with the face from image 2. Preserve lighting/background.
     ```
     - Structure: "Keep [list unchanged elements]. Change [edits]." E.g., for refinement: "Keep facial features, eyes, skin. Enhance details, sharpen eyes/lips."[1][2]
   - Settings: Resolution 1024-2048px (adjust ratio dropdown), steps 20-30, CFG 7-9.[2][3]

3. **Inpainting & Conditioning**:
   - Use **Inpaint Model Conditioning** node to restrict edits to masked face area (prevents body changes).[3]
   - Add **ControlNet** for pose estimation from ref image 3.[1][3]
   - Generate base edit output.

4. **Face Refinement Pass with FaceDetailer**:
   - Pipe output to **FaceDetailer** node:
     - Detects/crops face region.
     - Applies enhancement (inpainting + LoRA).
     - Pastes refined face back.[1]
   - **LoRA Integration** (face consistency Lightning LoRA):
     - Strength: 0.6-0.9 (increase if face drifts; decrease if over-stylized).[1]
     - Denoise: 0.3-0.5 (higher for strong corrections, lower for subtle).[1]
   - KSampler: kl_optimal scheduler, 2048x2048 for hi-res fix.[3]

5. **Upscaling & Final Polish**:
   - Use **SeedVR2** or Ultimate SD Upscale for 4K without face loss (e.g., 3801x3801).[1][3]
   - Optional: Repeat FaceDetailer for extra sharpness.

#### Tuning Tips
| Parameter | Face Drifts | Over-Stylized | Subtle Fix |
|-----------|-------------|---------------|------------|
| **LoRA Strength** | ↑ 0.8-1.0[1] | ↓ 0.4-0.6[1] | 0.5-0.7 |
| **Denoise** | 0.4-0.6[1] | 0.2-0.4[1] | 0.2-0.3 |
| **Prompt Weight** | Add "perfect face match"[2] | "Natural refinement"[2] | "Preserve identity"[1] |

This achieves **near-100% consistency** even in new scenes/poses.[1][3] Test iteratively; GGUF reduces VRAM vs FP16.[2] For 2025 updates, check ComfyUI docs for Qwen-2511 optimizations.[4]
